# Forex-AI-Prompt-Library

Forex AI Prompt Library
Institutional-grade prompt engineering workflows for macroeconomic analysis, liquidity tracking, and quantitative risk management in the Foreign Exchange markets.

The Problem Statement: Addressing the Liar’s Paradox in AI Trading
In the current volatile financial landscape, the primary barrier to AI integration is the "Liar’s Paradox"—the tendency for Large Language Models (LLMs) to generate hallucinated financial data or incorrect sentiment bias when processed via generic, low-context prompts.

In Forex trading, a 1% error in sentiment interpretation—such as misidentifying a "Hawkish" Federal Reserve stance as "Dovish"—leads to immediate capital erosion. Most retail-level prompts lack the deterministic guardrails required for institutional decision-making. This library solves that deficit by applying rigorous financial logic and structured prompting to ensure every model output is data-backed, verifiable, and execution-ready.

The Core Framework: The C.R.E.A.D. Formula
Every project in this repository is built using the C.R.E.A.D. architecture, a proprietary framework designed to eliminate model drift and ensure consistent output quality.

Context: Establishing the specific market environment (e.g., Post-NFP volatility, FOMC blackout periods).

Role: Assigning a high-authority institutional persona (e.g., Chief Risk Officer, Tier-1 Bank Analyst).

Execution: Defining the specific analytical task or entry trigger required.

Attribute: Setting the tonal and technical constraints (e.g., Data-driven, bearish/bullish bias, zero speculation).

Deliverable: Specifying the technical format (e.g., JSON Schema, Markdown tables, PineScript code).

Technical Specifications
To maintain institutional standards, the workflows in this repository utilize the following advanced Prompt Engineering techniques:

Deterministic Output Constraints: Utilizing system-level instructions to prevent the model from speculating on unconfirmed data.

Zero-Shot & Few-Shot Learning: Strategic use of examples within the prompt to prime the model for specific financial terminology.

JSON Schema Validation: Ensuring all outputs are formatted for programmatic consumption by trading bots or analytical software.

Negative Prompting: Explicitly defining "No-Go" zones to prevent common LLM pitfalls in financial forecasting.
